Final Year Project (FYP) - UCSI University

PROJECT TITLE
UniJobLink: UCSI University Internship Portal Platform

================================================================================
PROJECT DESCRIPTION
================================================================================

UniJobLink is a comprehensive web-based internship job platform designed
specifically for UCSI University students and partner companies. The platform
serves as a centralized hub where:

  • Students can discover, apply for, and manage internship opportunities
  • Companies can post job listings, review applications, and manage their
    recruitment pipeline
  • Administrators can oversee the entire ecosystem, verify companies, and
    monitor placement statistics

The platform streamlines the internship recruitment process for UCSI by
replacing manual processes with an automated, user-friendly digital solution.

KEY FEATURES
================================================================================

STUDENT FEATURES:
  • Job Discovery & Search
    - Browse available internship listings filtered by category, location,
      allowance, and duration
    - Save favorite jobs for later review
    - View detailed job information including requirements and company details
    - One-click application submission

  • Application Management
    - Track all applications with real-time status updates (PENDING, ACCEPTED,
      OFFERED, HIRED, REJECTED)
    - Distinguish between active applications and application history
    - Receive instant notifications when application status changes
    - Celebrate achievements with visual feedback when hired

  • Messaging & Communication
    - Real-time chat with recruiting companies
    - Per-conversation unread message badges
    - Sidebar notification badge showing total unread messages

  • Profile Management
    - Complete student profile with education, skills, and contact information
    - LinkedIn, GitHub, and portfolio URL integration
    - Professional resume upload capability

COMPANY FEATURES:
  • Job Posting Management
    - Create, edit, and archive job listings
    - Specify position requirements, allowance, duration, and available slots
    - Track application count per job in real-time
    - Switch between grid and list view for job management

  • Applicant Pipeline Management
    - Comprehensive ATS (Applicant Tracking System) view
    - Drag-and-drop style pipeline stages (Pending, Interviewing, Offered, Hired)
    - View applicant profiles and resume snapshots
    - Filter applications by job
    - Accept/offer/hire/reject applicants with automated notifications

  • Messaging & Collaboration
    - Direct communication with student applicants
    - Real-time chat thread per application
    - Notification badge for new applicants and messages

  • Dashboard Analytics
    - Key metrics: new applicants, active job listings, placements
    - Quick access to recent applications and job postings

ADMIN FEATURES:
  • Company Verification
    - Review and approve pending company registrations
    - Manage company verification status
    - Badge system showing pending verification count

  • User Management
    - Reset student and company passwords
    - Deactivate user accounts
    - View user management dashboard

  • System Overview
    - Dashboard with key statistics (total students, verified companies,
      placements)
    - Student placement tracker
    - Access to all administrative tools

TECHNICAL ARCHITECTURE
================================================================================

FRONTEND (React 19 + Vite)
  • Framework: React 19 with hooks for state management
  • Build Tool: Vite (ultra-fast ES module bundler)
  • Styling: Tailwind CSS (utility-first CSS framework)
  • Real-time: Socket.io client for messaging
  • Icons: Lucide React icon library
  • UI Components: Custom card, button, badge, modal, and form components
  • Responsive Design: Mobile-first approach with 768px breakpoint

BACKEND (Node.js + Express + Sequelize)
  • Runtime: Node.js with Express.js server framework
  • Database ORM: Sequelize (SQL query builder)
  • Authentication: JWT (JSON Web Tokens) with role-based access control
  • Real-time: Socket.io for instant messaging
  • File Upload: Multer middleware for resume and profile image uploads

DATABASE (MySQL)
  • Relational database with normalized schema
  • Core tables: Users, Students, Companies, JobPostings, Applications,
    Messages, ChatRooms
  • Tracking fields: Timestamps, seen/unseen status, application status
  • Indexing: Optimized queries for application filtering and search

DEPLOYMENT
  • Frontend: Vite dev server (localhost:5173)
  • Backend: Express server (localhost:5000)
  • Database: MySQL local/cloud instance

DATABASE SCHEMA HIGHLIGHTS
================================================================================

Users Table: Base user model (email, password, role: STUDENT/COMPANY/ADMIN)
Students Table: Student-specific data (firstName, lastName, faculty, skills, etc.)
Companies Table: Company information (companyName, industry, description, etc.)
JobPostings Table: Job listings with requirements and metadata
Applications Table: Student-to-job applications with status tracking
Messages Table: Individual messages with read/sender tracking
ChatRooms Table: Conversation groups between student and company

Key Fields for Notifications:
  - Application.companySeenAt / studentSeenAt: Tracks when unseen updates clear
  - Message.readAt: Tracks per-message read status
  - ChatRoom.lastMessage: Retrieves latest message for list display

================================================================================
USER ROLES & ACCESS CONTROL
================================================================================

STUDENT ROLE:
  - Access: /student/* routes
  - Capabilities: Apply for jobs, manage applications, message companies,
    edit profile
  - Restrictions: Cannot post jobs, cannot approve applications

COMPANY ROLE:
  - Access: /company/* routes
  - Capabilities: Post/edit/archive jobs, manage applications, message students,
    view analytics
  - Restrictions: Cannot apply for jobs, limited to own job postings

ADMIN ROLE:
  - Access: /admin/* routes
  - Capabilities: Verify companies, manage users, view system-wide analytics
  - Restrictions: Read-only for most data (except user password reset)

GUEST (Unauthenticated):
  - Access: /login and /register routes only
  - Can create new student or company account

SUMMARY
================================================================================

UniJobLink is a fully functional, production-ready internship management
platform that bridges the gap between UCSI University students and recruiting
companies. With a modern tech stack (React + Node.js + MySQL), responsive
design supporting mobile and desktop users, and robust features for real-time
communication and application tracking, the platform successfully automates
and streamlines the university's internship recruitment process.

The recent implementations focus on mobile accessibility and user engagement
through notifications and real-time messaging, making the platform accessible
to students and companies regardless of device or location.
